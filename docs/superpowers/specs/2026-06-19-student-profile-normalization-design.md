# 학생 정보 입력 정규화 설계

## 목적

학생이 학년, 반, 조, 번호를 입력할 때 띄어쓰기나 단위 글자의 오탈자가 있어도 숫자를 기준으로 표준 형식으로 저장한다. 관리자 제출자 카드에는 학생을 빠르게 식별할 수 있는 코드를 표시한다.

## 입력 정규화 규칙

공통 입력 정규화 모듈에서 각 입력값의 첫 번째 연속된 숫자를 추출한다.

- 학년: `1학년`, `2학년`, `3학년` 중 하나로 저장한다.
- 반: 양의 정수에 `반`을 붙여 `1반`, `2반`, `3반` 형식으로 저장한다.
- 조: 양의 정수에 `조`를 붙여 `1조`, `2조`, `3조` 형식으로 저장한다.
- 번호: 양의 정수에 `번`을 붙여 `1번`, `2번`, `3번` 형식으로 저장한다.
- 앞뒤 공백, 중간 띄어쓰기, `학녀`, `바`, `조오`처럼 단위 글자에 오탈자가 있어도 숫자가 유효하면 허용한다.
- 선행 0은 제거한다. 예를 들어 `01 반`은 `1반`, `003번`은 `3번`으로 저장한다.

## 유효성 검사

- 학년은 1~3만 허용한다.
- 반, 조, 번호는 1 이상의 정수만 허용한다.
- 숫자가 없거나 허용 범위를 벗어나면 세션을 만들지 않고 해당 항목을 확인하라는 오류 메시지를 표시한다.
- 이름은 기존처럼 앞뒤 공백만 제거한다.

## 관리자 카드 표기

관리자 제출자 명단 카드의 주 표시는 다음 식별 코드와 이름을 연결한다.

`학년 숫자 + 반 숫자 + 학생 번호 두 자리 + 학생 이름`

예시:

- 1학년 1반 1번 홍길동 → `1101홍길동`
- 2학년 3반 12번 김철수 → `2312김철수`

학생 번호는 두 자리보다 짧을 때 앞에 0을 붙인다. 반 번호는 입력된 숫자를 그대로 사용한다. 조는 카드 보조 정보에 표준화된 `N조` 형식으로 표시한다.

## 구현 경계

- 입력 정규화와 관리자 식별 코드 생성을 독립적인 순수 함수로 분리한다.
- 학생 시작 처리에서는 정규화된 값만 Firebase 세션과 제출물에 전달한다.
- 기존 데이터는 일괄 변경하지 않는다. 기존 관리자 카드에서도 가능한 경우 숫자를 추출해 동일한 식별 코드 형식으로 표시한다.
- 퀴즈 출제, 그림 제출, Firebase 권한 및 Firestore 규칙은 변경하지 않는다.

## 테스트

자동 테스트로 다음을 검증한다.

- 띄어쓰기와 단위 오탈자가 포함된 입력의 정상 변환
- 학년 1~3 범위 검사
- 반, 조, 번호의 양의 정수 검사
- 선행 0 제거
- 관리자 카드 식별 코드 생성과 학생 번호 두 자리 채움
- 기존 관리자 필터와 번호순 배치 동작 유지
