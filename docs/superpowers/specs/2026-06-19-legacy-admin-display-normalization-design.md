# 기존 데이터 관리자 표시 정규화 설계

## 목적

Firestore에 이미 저장된 학생 정보는 수정하지 않으면서, 관리자 제출자 화면에서는 신규 데이터와 동일한 학년·반·조·번호 형식으로 표시하고 분류한다.

## 표시 정규화

관리자 화면이 참여 기록을 구성할 때 기존 데이터의 각 필드에서 첫 번째 연속 숫자를 추출한다.

- 학년: 숫자가 1~3이면 `N학년`으로 표시한다.
- 반: 양의 정수이면 `N반`으로 표시한다.
- 조: 양의 정수이면 `N조`로 표시한다.
- 번호: 양의 정수이면 `N번`으로 표시한다.
- 선행 0은 제거한다.
- 숫자를 식별할 수 없거나 학년이 범위를 벗어나면 해당 필드의 기존 원문을 앞뒤 공백만 제거하여 유지한다.
- 학생 이름은 앞뒤 공백만 제거한다.

## 관리자 카드

표준화된 값을 사용해 카드 식별자를 생성한다.

`학년 숫자 + 반 숫자 + 학생 번호 두 자리 + 학생 이름`

예: 기존 값이 `1 학년`, `1 바`, `1 조`, `01`, `홍길동`이면 관리자 화면에는 다음처럼 나타난다.

- 카드 제목: `1101홍길동`
- 보조 정보: `1학년 1반`, `1번 · 1조`

## 관리자 기능 적용 범위

표준화된 관리자 화면용 기록을 다음 기능에 공통으로 사용한다.

- 학년 및 반 필터 항목
- 검색
- 전체 배치의 학년·반 구역
- 조별 배치
- 번호순 배치
- 카드 제목과 보조 정보
- 삭제 확인용 학생 정보

답안 팝업 열기, 공개 여부 변경, 삭제 등에 사용하는 세션 ID와 원본 제출물 연결은 그대로 유지한다.

## 데이터 보존

- Firestore의 `sessions`, `submissions`, `feedback` 문서는 변경하지 않는다.
- 관리자 로그인이나 화면 진입 시 데이터 쓰기를 실행하지 않는다.
- 신규 입력 정규화 기능은 기존대로 유지한다.

## 구현 구조

- `profile-utils.js`에 관리자 표시 전용 필드 정규화 함수와 프로필 정규화 함수를 추가한다.
- `admin-view-model.js`의 참여 기록 그룹 생성 단계에서 세션과 제출물의 학생 정보를 표시용으로 정규화한다.
- `admin-card-list.js`는 정규화된 기록만 받아 카드와 필터를 구성한다.

## 테스트

- 기존 오탈자 및 띄어쓰기 데이터가 표준 형식으로 변환되는지 검증한다.
- 선행 0이 제거되는지 검증한다.
- 숫자가 없는 기존 필드는 원문을 유지하는지 검증한다.
- 변환된 반과 조를 기준으로 필터 및 조별 구역이 합쳐지는지 검증한다.
- 변환된 번호를 기준으로 숫자순 정렬되는지 검증한다.
- 세션 ID와 제출물 배열이 변환 후에도 유지되는지 검증한다.
