# 관리자 제출 조회·캔버스 제출·답안 수정 설계

## 목표

원자 모형 퀴즈에 다음 기능을 추가한다.

- 관리자는 참여 기록별 제출자 명단을 보고, 학생 답안을 팝업에서 한 문제씩 조회한다.
- 학생은 사진 또는 캔버스 중 하나를 선택해 문제별 결과물을 제출한다.
- 학생은 현재 브라우저에서 자신의 제출 답안만 다시 조회한다.
- 관리자는 특정 답안에 수정 권한을 1회씩 부여한다.
- 학생은 부여된 횟수만큼 해당 답안의 결과물만 수정한다.
- Firebase Authentication과 Firestore Security Rules로 권한을 서버 측에서 강제한다.

## 인증과 권한

### 학생

- 앱 시작 시 Firebase 익명 인증을 사용한다.
- 익명 인증 UID를 세션과 각 제출 문서의 `ownerUid`에 저장한다.
- 학생은 현재 브라우저에서 생성한 `sessionId`만 앱 상태에 보관한다.
- 브라우저를 닫거나 다른 기기에서 접속한 뒤 학생 정보를 입력해 과거 기록을 찾는 기능은 제공하지 않는다.
- 학생은 자신의 `ownerUid`가 저장된 답안만 수정할 수 있다.

### 관리자

- 기존의 코드 내 고정 비밀번호를 제거한다.
- 관리자 로그인 화면에는 Gmail 아이디와 비밀번호만 입력한다.
- 아이디 입력값 뒤에 `@gmail.com`을 자동으로 붙여 Firebase 이메일·비밀번호 로그인을 수행한다.
- 사용자가 이메일 전체 주소를 입력해도 `@gmail.com`이 중복되지 않도록 정규화한다.
- 로그인한 사용자의 UID가 아래 값과 일치해야 관리자 기능을 허용한다.

  `E2aywHHeSvbpcVW81AxcYIrubvE2`

- 비밀번호는 코드, 로컬 저장소, Firestore에 저장하지 않는다.
- 관리자 로그인 상태는 Firebase Authentication이 관리한다.
- 관리자 로그아웃 버튼을 제공한다.

## Firestore 데이터 구조

### `sessions/{sessionId}`

기존 학생 정보와 함께 다음 필드를 저장한다.

- `ownerUid`: 익명 인증 UID
- `createdAt`
- `completedAt`

동일 학생이 여러 번 참여하면 각 `sessionId`를 별도 참여 기록으로 유지한다.

### `submissions/{submissionId}`

기존 필드와 함께 다음 필드를 저장한다.

- `ownerUid`: 제출 학생의 익명 인증 UID
- `submissionType`: `"photo"` 또는 `"canvas"`
- `imageData`: 압축된 JPEG 데이터 URL
- `editAllowance`: 남은 수정 횟수, 최초값 `0`
- `modifiedAt`: 마지막 수정 시각 또는 `null`

기존 문서에 `submissionType`이 없으면 화면에서는 `"photo"`로 처리한다. 기존 문서의 `editAllowance`가 없으면 `0`으로 처리한다.

### 피드백

- 피드백 문서는 기존 제출 문서 ID를 계속 참조한다.
- 답안 결과물이 수정되어도 제출 문서 ID를 바꾸지 않는다.
- 따라서 기존 피드백은 삭제하거나 이동하지 않고 그대로 유지한다.

## Firestore 보안 규칙

규칙은 다음을 서버 측에서 강제한다.

- 인증된 사용자만 세션·제출·피드백을 생성하거나 읽는다.
- 학생 세션과 제출 생성 시 `ownerUid == request.auth.uid`여야 한다.
- 일반 학생은 관리자 설정을 변경할 수 없다.
- 관리자 UID만 앱 설정을 변경하고 수정 권한을 부여할 수 있다.
- 관리자는 특정 제출 문서의 `editAllowance`를 정확히 1씩 증가시킬 수 있다.
- 제출 소유자는 `editAllowance > 0`일 때만 결과물을 수정할 수 있다.
- 학생 수정 시 `editAllowance`는 정확히 1 감소해야 한다.
- 학생 수정 시 변경 가능한 필드는 `imageData`, `submissionType`, `editAllowance`, `modifiedAt`으로 제한한다.
- 학생 정보, 문제 번호, 원소 번호, 빈칸 답안, 소유 UID, 세션 ID는 수정할 수 없다.
- 문서 삭제는 허용하지 않는다.

수정은 Firestore 트랜잭션으로 수행한다. 트랜잭션에서 현재 `editAllowance`를 다시 읽고, 결과물 교체와 1회 차감을 하나의 원자적 작업으로 처리한다.

## 관리자 화면

### 제출자 명단

- 제출 문서를 카드로 나열하지 않고 `sessionId` 기준 참여 기록 명단을 표시한다.
- 각 행에는 학년, 반, 조, 번호, 이름, 참여 시각, 제출 완료 수를 표시한다.
- 같은 학생이 여러 번 참여하면 참여 기록마다 별도 행으로 표시한다.
- 기존 검색 필터를 유지한다.

### 답안 팝업

- 참여 기록을 누르면 명단 위에 모달 팝업을 표시한다.
- 배경 스크롤을 잠근다.
- 한 페이지에는 한 문제의 다음 내용만 표시한다.
  - 학생이 제출한 사진 또는 캔버스 결과물
  - 해당 제출물에 달린 기존 피드백
- 정답 원자 모형과 정답 표는 표시하지 않는다.
- 상단에 학생 정보와 `현재 문제 / 전체 문제`를 표시한다.
- `이전`, `다음` 버튼과 좌우 스와이프로 문제를 이동한다.
- 첫 문제에서는 이전 버튼, 마지막 문제에서는 다음 버튼을 비활성화한다.
- 현재 답안에 `수정 권한 1회 부여` 버튼을 제공한다.
- 권한 부여 성공 후 남은 수정 횟수를 표시한다.
- 닫기 버튼, 팝업 바깥 영역, `Esc`, 스마트폰 브라우저 뒤로가기로 팝업을 닫는다.
- 팝업이 닫히면 기존 필터와 스크롤 위치를 유지한 제출자 명단으로 돌아간다.
- 팝업 열기·닫기는 History API 상태로 관리해 뒤로가기가 페이지 이탈보다 먼저 팝업을 닫도록 한다.

## 학생 퀴즈 화면

### 제출 방식 선택

- 원자 정보 표 바로 아래에 `사진 촬영/선택`과 `캔버스에 그리기` 선택 버튼을 둔다.
- 한 문제에서 두 방식 중 하나만 활성화한다.
- 제출 시 활성화된 방식의 결과물 하나가 반드시 있어야 한다.
- 방식 변경 시 다른 방식의 임시 결과물은 제출 대상에서 제외한다. 실수로 전환한 경우를 대비해 다시 돌아오면 작성 중 결과는 현재 문제를 벗어나기 전까지 유지한다.

### 사진

- 모바일에서 촬영 또는 갤러리 선택이 가능하도록 이미지 파일 입력을 사용한다.
- 파일을 선택하거나 촬영하면 즉시 현재 퀴즈 페이지에 미리보기를 표시한다.
- 제출 전에 기존 이미지 압축 절차를 적용한다.

### 캔버스

- 일반 퀴즈 화면에서 캔버스를 바로 표시한다.
- 일반 화면의 기본 도구는 `검정 펜`이며 즉시 그릴 수 있다.
- 도구 모음은 다음 순서로 제공한다.
  - 검정 펜
  - 전자
  - 양성자
  - 중성자
  - 지우개
  - 선택/이동
  - 전체화면
- 검정 펜은 검정색 자유 선을 그린다.
- 전자, 양성자, 중성자는 서로 구분되는 고정 색상의 펜으로 그린다.
- 지우개는 드래그 경로를 지운다.
- 선택/이동은 그리기 대신 캔버스 시점을 이동한다.
- 전체화면에 들어가면 기본 도구를 `선택/이동`으로 변경한다.
- 전체화면에서는 드래그로 시점을 이동하고, 그리기 도구를 선택하면 해당 도구로 다시 그린다.
- 전체화면을 닫아도 그림과 시점은 유지한다.
- 캔버스는 내부 작업 영역을 고정 크기로 유지하고 CSS 변환으로 시점만 이동한다.
- 제출 시 보이는 화면이 아니라 전체 작업 영역을 흰 배경 JPEG로 내보낸다.
- 사진과 동일한 최대 크기와 JPEG 압축 검사를 적용한다.
- 빈 캔버스는 제출할 수 없다.

## 학생 내 답안 화면

- 3문제 제출 완료 후 결과 화면 상단에 `내 답안 조회` 버튼을 표시한다.
- 현재 세션의 제출 답안 3개만 표시한다.
- 정답 원자 모형은 표시하지 않는다.
- 다른 학생 제출물이나 다른 학생용 피드백 입력 기능을 표시하지 않는다.
- 답안별 제출 결과물과 제출 유형을 조회할 수 있다.
- 화면 진입 시 Firestore에서 수정 권한을 확인한다.
- `권한 새로고침` 버튼을 제공해 학생이 수동으로 다시 확인할 수 있게 한다.
- `editAllowance > 0`인 답안에만 `수정하기` 버튼을 표시한다.
- 수정 화면은 퀴즈와 동일한 사진·캔버스 선택 UI를 사용한다.
- 수정 성공 후 화면 데이터를 다시 읽고, 권한이 모두 소진되면 수정 버튼을 숨긴다.
- 관리자에게 권한을 여러 번 부여받으면 남은 횟수만큼 반복 수정할 수 있다.

## 이미지 저장과 데이터베이스 용량

- 사진과 캔버스 결과는 모두 JPEG 데이터 URL로 저장한다.
- 기존 `900px`, JPEG 품질 `0.68`, 최대 약 `700KB` 검사를 공통으로 적용한다.
- 캔버스 결과도 같은 이미지 유틸리티를 거쳐 크기를 제한한다.
- Firestore 문서 제한에 근접하지 않도록 압축 실패 또는 최대 크기 초과 시 저장하지 않고 학생에게 다시 선택하거나 그림을 단순화하도록 안내한다.

## 코드 구조

- `src/app.js`: 화면 전환과 기능 조합
- `src/firebase-service.js`: 인증, 조회, 관리자 권한 부여, 답안 수정 트랜잭션
- `src/drawing-canvas.js`: 캔버스 도구, 포인터 입력, 이동, 전체화면, 이미지 내보내기
- `src/submission-editor.js`: 사진·캔버스 제출 방식의 공통 상태와 유효성 검사
- `src/image-utils.js`: 사진 및 캔버스 JPEG 압축·크기 검사
- `firestore.rules`: 인증과 필드 단위 수정 권한 강제
- `styles.css`: 모달, 명단, 제출 선택기, 캔버스, 전체화면 스타일

`app.js`에 캔버스 포인터 처리와 제출 편집 상태를 직접 추가하지 않고 독립 모듈로 분리한다.

## 오류 처리

- 인증 실패 시 퀴즈 시작이나 관리자 기능을 진행하지 않고 재시도 안내를 표시한다.
- 관리자 UID가 다르면 즉시 로그아웃하고 관리자 접근을 거부한다.
- Gmail 아이디가 비어 있거나 잘못된 경우 로그인 요청 전에 안내한다.
- 권한 부여 중 문서가 사라졌거나 통신이 실패하면 횟수를 화면에 미리 반영하지 않는다.
- 학생 수정 중 다른 요청으로 권한이 소진되면 트랜잭션을 실패시키고 새로고침을 안내한다.
- 이미지 압축·캔버스 변환·Firestore 저장 실패 시 기존 답안은 그대로 유지한다.
- 브라우저가 Fullscreen API를 지원하지 않으면 화면을 채우는 고정 레이어 방식으로 대체한다.

## 테스트

- Gmail 아이디 정규화와 관리자 UID 검증
- 참여 기록별 명단 그룹화와 정렬
- 팝업에 제출물·피드백만 표시되고 정답이 제외되는지 확인
- 팝업 이전·다음 경계와 History API 닫기 처리
- 사진 선택 즉시 미리보기
- 제출 방식 하나만 활성화되고 결과물이 필수인지 확인
- 일반 캔버스 기본 도구가 검정 펜인지 확인
- 전체화면 진입 시 선택/이동으로 바뀌는지 확인
- 도구별 그리기, 지우기, 시점 이동, 빈 캔버스 검사
- 캔버스 JPEG 내보내기와 크기 제한
- 수정 권한이 없는 답안에는 수정 버튼이 표시되지 않는지 확인
- 새로고침 후 권한이 있는 답안에만 수정 버튼이 표시되는지 확인
- 수정 트랜잭션이 이미지 교체와 횟수 1 감소를 함께 수행하는지 확인
- 기존 피드백이 수정 후에도 같은 제출 문서에 남는지 확인
- Firestore Emulator로 학생·관리자·타 학생의 허용 및 거부 규칙 검증

- Authentication에서 `익명` 제공자 활성화
- Authentication에서 `이메일/비밀번호` 제공자 활성화
- 관리자 계정 생성
- 관리자 UID가 `E2aywHHeSvbpcVW81AxcYIrubvE2`인지 확인
- Authorized domains에 실제 GitHub Pages 도메인 등록
- 구현 후 새 `firestore.rules` 게시
